Inheritance in JS

Inheritance is, one Object getting access to the properties and methods of another Object. JS supports something called Prototypal Inheritance. This is much more simple, extensible and felxible when compared to Inheritance in other programming languages. Programming languages like C,C++ and Java supports Classical Inheritance.

Prototype

Objects are collection of key value pairs. They have user defined properties and methods and much more hidden or default created ones too. One such property is __proto__ which is called the Prototype. This property is available on all the Objects. The __proto__ Object can have its own properties and methods, which the parent Object can use and consume as, as it's own.

Prototype chain

var person = {
  name:'default'
  getname:function(){
    return this.name
  }
}

var shihab = {
  name:'Shihab'
}

shihab.__proto__ = person;

In the above code, if I do shihab.getname(), the engine try to find getname() in shihab Object. As it wasn't found it will move to shihab.__proto__. As it is available over there, it stops and uses that method. This is called Prototypal Chain. This comes into play for Objects. __proto__ can inturn have its own __proto__ and this can go on. It goes in search of property and method until the __proto__ property returns null.

As it is evident from the example, no one has to mention object_name.__proto__.property_name to find the properties (or methods). object_name.property_name is enough and rest engine will take care. So from outside it is like, that particular method or property is within the parent Object, even though it is referred via it's prototype.

In JS all are Objects or Primitives. Everything have a prototype by default. The inbuilt methods are made available to them through this __proto__ Object.
